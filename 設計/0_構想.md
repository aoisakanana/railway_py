# 概要  

pythonでrailwayな運用自動化ツールを作成する場合のフレームワークを作成します。  
手軽にrailwayパラダイムなプログラミング環境が整い、グラフノードごとの関数を実装するだけでrailwayになることが理想です。  
できればグラフノード単位でステートレスになるように設計できるとなおよいです。  
原則として関数型パラダイムを採用します。  

## 要件  

1. 複数エントリーポイントあり  
   例えば project_root/src/entry1.py project_root/src/entry2.py のような感じ。  
2. 共通ライブラリやユーティリティはcommonディレクトリに切り出し  
   project_root/src/common/api_caller.py のような感じ  
3. パッケージ管理は uv を使用  
   cd project_root 後、  
   ../uv/uv run python -m entry1 のようにして実行  
4. ユーザーは1コマンドでエントリーポイントを任意に追加  
   コマンド実行すると、新たなエントリーポイントとなるファイルが作成される。ファイルには必要なひな形が最初から作成されている。  
   Result型のハンドリングなどがエントリーポイントレベルで記載済み。  
   また、コマンドライン引数がプレースホルダで書かれており、そのコマンドラインをコンテキスト変数に渡すところや、コンテキスト変数を初期化するところなどが記載済みになっている。  
   また.envなどの外部設定ファイルから設定値を読み込めるようになっている。  
5. .env を使用して環境などを指定できるつくり。  
6. project_root/config で必要な設定をカスタムできる。  
   .env の環境設定値が development であれば、 project_root/config/development/ 配下の各種モジュール用設定ファイルを見に行くつくり  

## 仕様    

1. railwayのために returns を採用  
2. retryには tenacity を採用  
3. .env の読み込み、定数初期化は project_root/src/settings.py が実施。
